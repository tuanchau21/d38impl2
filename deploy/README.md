# Deployment artifacts — d38shop

Artifacts for deploying the bulk shoe shop to Oracle Cloud Always Free. See [../../plan/high-level-plan.md](../../plan/high-level-plan.md) and the [deployment plan](../../plan/deployment-plan.md) for full context.


| File                      | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **nginx-d38shop.conf**    | Nginx reverse proxy: `/api` → backend (port 3001), all other paths → frontend (port 3000). Optional: SSL, basic auth or IP allowlist for `/admin` and `/api/admin`.                                                                                                                                                                                                                                                                                                                                                                                 |
| **.env.example**          | Index: points to `.env.backend.example` and `.env.frontend.example`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **.env.backend.example**  | Backend env template (DB, Object Storage, `ADMIN_API_KEY`). Copy to `backend/.env` on the server.                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **.env.frontend.example** | Frontend env template (`API_URL` or `NEXT_PUBLIC_API_URL` for SSR). Copy to `frontend/.env` (optional).                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **schema.sql**            | MySQL HeatWave schema: `categories`, `products`, `product_images`, plus placeholder tables. Run once per database. Used by the backend.                                                                                                                                                                                                                                                                                                                                                                                                             |
| **deploy.sh**             | Builds and starts both apps: from repo root run `deploy/deploy.sh`. Runs `npm ci` and `npm run build` in `backend` and `frontend`, then restarts `d38shop-backend` and `d38shop-frontend` via PM2.                                                                                                                                                                                                                                                                                                                                                   |
| **seed-admin.js**         | One-time seed: create first admin from env `ADMIN_SEED_EMAIL`, `ADMIN_SEED_PASSWORD` (see script header). Run from repo root after `cd deploy && npm install` (once).                                                                                                                                                                                                                                                                                                                                                                                 |
| **manage-admin-users.js** | CLI to create, edit, or remove admin users. DB from `backend/.env` (MYSQL_HOST, MYSQL_USER, MYSQL_PASSWORD, MYSQL_DATABASE). **Create:** `node deploy/manage-admin-users.js create --email admin@example.com --password secret` (optional: `--name`, `--role`). **Edit:** `node deploy/manage-admin-users.js edit --email admin@example.com` (optional: `--password`, `--name`, `--role`). **Remove:** `node deploy/manage-admin-users.js remove --email admin@example.com` or `--id 1`. Run from repo root after `cd deploy && npm install` (once). |


## Quick usage

1. **Provision:** Create MySQL HeatWave, Object Storage bucket + credentials, and Compute VM (same region).
2. **VM:** Install Node.js LTS, Nginx, PM2. Create app dir (e.g. `/var/www/d38shop`), clone repo. Copy `.env.backend.example` to `backend/.env` and `.env.frontend.example` to `frontend/.env` (optional); fill values for each app.
3. **Nginx:** Include or copy `nginx-d38shop.conf`; set `server_name` and reload Nginx.
4. **Database:** Run `schema.sql` against the app database (e.g. `mysql -h HOST -u USER -p DATABASE < deploy/schema.sql` from repo root).
5. **Deploy:** From repo root run `deploy/deploy.sh` to build and start both frontend and backend.
6. **Deploy scripts (seed-admin, manage-admin-users):** From repo root run `cd deploy && npm install` once, then use `node deploy/seed-admin.js` (env vars) or `node deploy/manage-admin-users.js create|edit|remove ...`.
